Remote Access to Local Storage (RATLS)

Feature doesn’t work
    Check for:
  • Customer is not on v2.12+ of the app
  • “Record Locally” isn’t enabled
  • Direct Storage Access isn’t enabled
  • SmartHub is offline
  • iOS or Android device is unable to connect to the internet
  • Storage was removed from the SmartHub, or otherwise corrupted
  • Customer does not have “Overwrite Automatically” enabled for Local Storage, and the storage device ran out of space (a push notification is sent if storage is getting low and overwrite is not selected, however)
  • Inside Home connection: Customer is not on the same network as router/SmartHub
Feature doesn’t work outside the home connection
    Check for:
  • Customer don’t have VPN configured on router and mobile device correctly
    • Advise customer to check with router manufacturer and the provider of the VPN app you are using
  • There is a mismatch between the Port Forwarding settings in the app vs router; check both and ensure they match
    • Check with router manufacturer for where to find, and change, Port Forwarding settings
  • There was a power failure on the router
    • VPN as a result may need to be checked
    • Port forwarding settings may need to be checked
    • User should confirm no mismatch between Port Forwarding settings in app vs router
    • User should also, in their router, assign a static DHCP to the SmartHub. This means that, even if there is a power failure, the SmartHub will always have the same IP address
  • User has two routers, where the SmartHub-connected router is also connected to a router
    • Check with your router manufacturers for where to find, and change, Port Forwarding settings and resolve the issue for both routers
  • Owner of the Wi-Fi network does not allow the non-standard port we use for port forwarding
    • Check with the owner of the Wi-Fi network
  • The user’s home ISP does not allow port forwarding
    • Check with the ISP
